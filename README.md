# Government Digital and Data → SFIA 9 mapping repo

This repository package is a handover/starter repo for the DDaT-to-SFIA mapping work completed in the working session.

## Purpose

The project maps:

```text
Role family → Role → Role level → Government capability → SFIA skill → recommended SFIA level
```

The latest product direction is the **collapsible graph/tree** where role families, roles, capabilities and SFIA skills are expandable, and recommended SFIA levels can be highlighted or filtered.

## Quick start

1. Create and activate a Python environment.

```bash
python -m venv .venv
source .venv/bin/activate      # macOS/Linux
# .venv\Scripts\activate     # Windows
pip install -r requirements.txt
```

2. Put the raw inputs into `data/raw/`.

Expected filenames:

```text
data/raw/Role and skill content - Capability Framework - Government Digital and Data profession2026-06-05_11-47-23.csv
data/raw/sfia-9_current-standard_en_260521.xlsx
```

3. Build processed data and the HTML explorer.

```bash
python scripts/build_all.py
```

4. Validate outputs.

```bash
python scripts/validate_outputs.py
```

5. Open:

```text
web/collapsible_role_capability_skill_level_tree.html
```

## Current state

The mapping is **first-pass resolved**:

- 185 Government capabilities resolved in the working session/output set.
- 52 roles and 205 role-level nodes in the source data.
- Role-level → SFIA-level recommendations are heuristic and should be SME-reviewed before formal use.

## Key files

- `mapping/mapping_decisions.json` — capability → SFIA mapping decisions.
- `mapping/level_calibration.json` — role-level and Government skill-level heuristic for SFIA level recommendations.
- `scripts/build_all.py` — build script to generate CSV, JSON, workbook, and HTML outputs.
- `scripts/validate_outputs.py` — lightweight output validation.
- `docs/data_model.md` — graph schema.
- `docs/mapping_methodology.md` — mapping and level-calibration method.
- `docs/artifact_manifest.md` — list of generated artefacts from the working session.

## Important notes

- `BUAN` and `TESTA` were discussed during review but were not present in the uploaded SFIA 9 workbook. This repo uses `BUSA` instead of `BUAN`, and `TEST/NFTS/PROG` instead of `TESTA`.
- Generated files are ignored by default in `.gitignore`; if you want to commit outputs, adjust `.gitignore`.
- The D3 web products generated by the script load D3 from a CDN. For offline operation, vendor D3 into `web/vendor/` and update the generated HTML template.
